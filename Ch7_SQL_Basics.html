What is SQL?

SQL stands for Structured Query Language. It is used to create, remove, alter the database and database objects, and to store, retrieve, update data in a database. It is the standard language for creating, accessing, and manipulating DBMS.

Concept of SQL

‚ë† User specifies a condition ‚Üí ‚ë° Program searches all records in the database ‚Üí ‚ë¢ Selects records satisfying the condition ‚Üí ‚ë£ Returns statistical information ‚Üí ‚ë§ Result is stored in the form of a table.

Chapter 7 ¬∑ Section 1

Types of SQL Commands

SQL commands are grouped into 5 categories based on their purpose. The most important distinction: DDL is auto-committed, DML is not.

DDL

Data Definition Language

‚úì Auto-committed

Defines & modifies database schema. Changes are permanent ‚Äî cannot be rolled back.

  • CREATE ‚Äî create table/database
  • ALTER ‚Äî modify table structure
  • TRUNCATE ‚Äî delete all records
  • DROP ‚Äî remove table/database
  • RENAME ‚Äî rename a table
DML

Data Manipulation Language

‚úó Not Auto-committed

Manipulates data inside tables. Changes can be rolled back before COMMIT.

  • INSERT ‚Äî insert a new row
  • UPDATE ‚Äî update existing row
  • DELETE ‚Äî delete a row
  • MERGE ‚Äî merge two rows/tables
DCL

Data Control Language

‚úì Auto-committed

Provides commands to grant and take back user authority/permissions.

  • GRANT ‚Äî grant permission
  • REVOKE ‚Äî take back permission
TCL

Transaction Control Language

Manual control

Keeps check on other commands. Can annul changes by rolling back or make them permanent.

  • COMMIT ‚Äî permanently save
  • ROLLBACK ‚Äî undo change
  • SAVEPOINT ‚Äî save temporarily
DQL

Data Query Language

Read-only

Most commonly used. Retrieves data ‚Äî complete tables or partial using WHERE clause.

  • SELECT ‚Äî extract data
  • SELECT DISTINCT ‚Äî unique values only
DDL ‚Äî Auto-committed (Permanent)

All DDL commands save changes permanently the moment they are executed. There is NO way to undo them with ROLLBACK. Once you DROP a table ‚Äî it is gone forever.

DML ‚Äî NOT Auto-committed (Reversible)

DML changes are not permanent until you COMMIT. If you make a mistake with INSERT/UPDATE/DELETE, you can use ROLLBACK to undo. Flow: DML ‚Üí ROLLBACK (undo) OR COMMIT (make permanent).

Chapter 7 ¬∑ Section 2

Data Definition Language (DDL)

DDL commands define and modify the structure (schema) of database objects. All DDL commands are auto-committed.

CREATE ‚Äî Creating a Table

Used to create a new table. You define the table name, columns, and their data types.

Syntax:
CREATE TABLE tablename ( column1 datatype(size), column2 datatype(size), ... );
Example ‚Äî Customer Table (used in all SELECT examples):
CREATE TABLE Customer ( CustomerID number, CustomerName char(20), ContactNo number, address varchar(20), city char(20), Postalcode varchar(20), country char(20) );

Creates Customer table with 7 columns ‚Äî this table is used throughout the chapter for examples.

Chapter 7 ¬∑ Section 3

ALTER Command

ALTER is used for alteration of existing table structures. It has 5 sub-operations ‚Äî click each tab to explore.

To Add a Column to an Existing Table:
ALTER TABLE table-name ADD(column-name datatype);
Example
Add an address column (varchar 50) to the Students table.
ALTER TABLE Students ADD(address varchar(50));

This command adds a new column address to the Students table. Existing rows get NULL in this new column.

To Add Multiple Columns at Once:
ALTER TABLE table-name ADD(col1 type1, col2 type2);
Example
Add dob and city columns to students table in one command.
ALTER TABLE students ADD(dob number(10), city varchar(10));
To Add a Column with a Default Value:
ALTER TABLE table-name ADD(col-name datatype DEFAULT data);
Example
Add a branch column with default value 'COE' to students.
ALTER TABLE students ADD(branch varchar(10) DEFAULT 'COE');

Any new row inserted without specifying branch will automatically get 'COE' as its branch value.

To Modify Data Type / Size of an Existing Column:
ALTER TABLE table-name MODIFY(column-name datatype);
Example
Change the address column from varchar(50) to char(30).
ALTER TABLE students MODIFY(address char(30));

Modifies the address column data type in the Students table.

To Rename an Existing Column:
ALTER TABLE table-name RENAME COLUMN old-name TO new-name;
Example
Rename the address column to Location in Student table.
ALTER TABLE Student RENAME COLUMN address TO Location;

The above command renames the address column to Location.

To Drop (Remove) a Column:
ALTER TABLE table-name DROP(column-name);
Example
Permanently remove the address column from Students table.
ALTER TABLE Students DROP(address);

This permanently removes the column and all its data.

TRUNCATE Command

TRUNCATE removes all records from a table but does NOT destroy the table structure. When applied, the Primary Key counter is also reset (re-initialized).

Syntax:
TRUNCATE TABLE table-name;
Example
TRUNCATE TABLE students;

Deletes all records from students table. The table itself (columns, structure, constraints) remains intact and PK resets to start from 1.

DROP Command

DROP completely removes a table (or database) including its structure, all data, and all associated constraints. This is irreversible.

Syntax:
DROP TABLE table-name; DROP DATABASE database-name;
Examples
DROP TABLE students; -- removes table completely DROP DATABASE Test; -- removes database named Test

RENAME Command

Used to rename an entire table.

Syntax:
RENAME old_table_name TO new_table_name;
Example
RENAME students TO people;

Renames the Students table to People.

Chapter 7 ¬∑ Section 4

Data Manipulation Language (DML)

DML commands deal with the manipulation of data present in the database. They are NOT auto-committed ‚Äî changes can be rolled back.

DML commands: INSERT, SELECT, SELECT DISTINCT, UPDATE, DELETE

1. INSERT Command

Used to insert data (new rows) into a table.

Syntax:
INSERT INTO table-name VALUES(data1, data2, ...);
Example ‚Äî Students Table (Roll, Name, Age)
INSERT INTO Students VALUES(1, 'mani', 22); INSERT INTO Students VALUES(2, 'mahi', 24); INSERT INTO Students VALUES(3, 'appy', 20);
Inserting NULL value:

Use the keyword null to explicitly insert a NULL into a column.

INSERT INTO students VALUES(4, 'padm', null);

Row 4 will have NULL in the age column.

Inserting DEFAULT value:

Use the keyword default to use the column's predefined default value.

INSERT INTO students VALUES(5, 'rahul', default);

If age has default 18 ‚Üí row 5 gets age = 18.

2. UPDATE Command

Used to update/modify existing rows in a table. Always use WHERE clause to avoid updating ALL rows accidentally.

Syntax:
UPDATE table_name SET column1=value1, column2=value2, ..., columnN=valueN WHERE [condition];
Update single column
UPDATE students SET age=22 WHERE id=4;

Updates padm's age from NULL ‚Üí 22

Update multiple columns
UPDATE students SET name='pri', age=25 WHERE id=5;

Updates both name and age for id=5 in one query

Update CustomerName to 'abc' where City='Berlin.' AND Country='Germany'.

3. DELETE Command

DELETE removes rows from a table. Can be used with or without WHERE clause. Unlike TRUNCATE, it is DML and can be rolled back.

Syntax:
DELETE FROM table-name; -- deletes ALL rows DELETE FROM table-name WHERE condition; -- deletes specific rows
Delete all records
DELETE FROM students;

Deletes ALL records from students table. Table structure remains.

Delete specific record
DELETE FROM students WHERE id=5;

Deletes only the row where id = 5.

Comparison Operators

SQL provides powerful comparison operators to filter data in WHERE clauses: IN, BETWEEN, and LIKE.

IN Operator

Allows specifying multiple values in a WHERE clause. Shorthand for multiple OR conditions.

SELECT col FROM table WHERE col IN (val1, val2, val3);

BETWEEN Operator

Selects values within a given range. Values can be numbers, text, or dates.

SELECT col FROM table WHERE col BETWEEN val1 AND val2;

LIKE Operator

Used to search for a specified pattern in a column using wildcards % and _.

SELECT col FROM table WHERE col LIKE 'pattern';
