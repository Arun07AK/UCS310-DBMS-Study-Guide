Section 01

What is a Key?

Definition

  • A DBMS key is an attribute or set of attributes which helps you to identify a row (tuple) in a relation (table)
  • Keys allow you to find the relation between two tables
  • Keys help you uniquely identify a row in a table by a combination of one or more columns

Why Do We Need Keys?

  • Help you identify any row of data in a table
  • Real-world tables could have thousands of records and records could be duplicated
  • Keys ensure you can uniquely identify a table record despite these challenges
  • Allows you to establish and identify relationships between tables
  • Help to enforce identity and integrity in the relationship

Super Key

Super Key
Super Key?

A superkey is a group of single or multiple keys which identifies rows in a table. A Super key may have additional attributes that are not needed for unique identification.

In this example, EmpSSN, EmpNum and Empname are superkeys:

Here: {EmpSSN}, {EmpNum}, {Empname}, {EmpSSN, EmpNum}, {EmpSSN, Empname}, {EmpNum, Empname}, {EmpSSN, EmpNum, Empname} ‚Äî ALL are superkeys.

Section 03

Primary Key

Primary Key
Primary Key?

A column or group of columns in a table which helps us to uniquely identify every row in that table. Cannot be duplicate ‚Äî same value can't appear more than once.

Rules for defining Primary Key:

  • Two rows can't have the same primary key value
  • It is mandatory for every row to have a PK value
  • The primary key field cannot be NULL
  • PK value can be modified only carefully (if FK references it)

Candidate Key

Candidate Key
Candidate Key?

A super key with no repeated attribute (i.e., minimal super key). The Primary Key should be selected from the candidate keys. Every table must have at least a single candidate key.

Properties of Candidate Key:

  • It must contain unique values
  • Candidate key may have multiple attributes
  • Must not contain null values
  • It should contain minimum fields to ensure uniqueness
  • Uniquely identify each record in a table

Example: StudID, Roll No, Email are ALL Candidate Keys

All three (StudID, RollNo, Email) uniquely identify each student ‚Üí ALL are candidate keys. One is chosen as PK (StudID), rest become Alternate Keys.

Composite Key

Composite Key
Composite / Compound Key?

‚Ä¢ A key which has multiple attributes to uniquely identify rows is called a Composite Key.
‚Ä¢ Type of Compound key ‚Äî has many fields to uniquely recognize a record.
‚Ä¢ It is possible that each column may NOT be unique by itself.
‚Ä¢ However, when combined, the combination becomes unique.

Example: OrderNo + ProductID = Composite Key

Surrogate / Artificial Key
Surrogate Key?

‚Ä¢ An artificial key which aims to uniquely identify each record is called a Surrogate Key.
‚Ä¢ Created when you don't have any natural primary key.
‚Ä¢ They do not lend any meaning to the data in the table.
‚Ä¢ Surrogate key is usually an integer.

Section 09

Difference: Primary Key vs Foreign Key

Primary KeyForeign Key
Helps to uniquely identify a record in the table.It is a field in the table that is the primary key of another table.
Primary Key never accepts null values.A foreign key may accept multiple null values.
PK is a clustered index. Data is physically organised in the sequence of the clustered index.FK cannot automatically create an index. However, you can manually create an index on the FK.
You can have only ONE primary key in a table.You can have multiple foreign keys in a table.

Flashcards

Super Key
May have extra attributes
Definition
Group of single or multiple keys that identifies rows. May have additional attributes NOT needed for uniqueness. All supersets of CKs are super keys.
Primary Key
4 rules to remember
4 Rules
1. No two rows same PK. 2. Every row MUST have PK. 3. PK cannot be NULL. 4. Modify only carefully if FK references it. Only ONE PK per table.
Candidate Key
Minimal super key
Definition
Super key with no repeated/extra attribute. Minimal. PK selected from CKs. Every table has at least ONE CK. Unique, NOT NULL, minimum fields.
Alternate Key
Which candidate keys?
Definition
All candidate keys that are NOT the primary key. StudID=PK ‚Üí Roll No & Email = Alternate Keys. Table can have multiple alternate keys.
Foreign Key
What does it reference?
Definition
Column added to create relationship with another table. References the PK of another table. Maintains data integrity. Multiple FKs allowed per table. Null values allowed.
Composite Key
How many attributes?
Definition
Key with MULTIPLE attributes. Neither attribute is unique alone, but their COMBINATION is unique. E.g., (OrderNo, ProductID) ‚Äî each repeats alone but combined = unique.
Surrogate Key
When is it needed?
Definition
Artificial key (usually integer) created when NO natural PK exists. No real-world meaning. E.g., Sr_No added to student table with duplicate names. Auto-generated.
Referential Integrity
Which key enforces this?
Definition
Enforced by the Foreign Key. FK value in child table MUST match an existing PK value in the parent table. This concept is also called Referential Integrity.
